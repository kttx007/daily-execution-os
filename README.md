# 人生乘法任务中枢 Daily Execution OS

一个面向个人长期复利的执行管理系统。它不是普通 Todo List，而是把每日收集、今日执行、四象限判断、未完成顺延、卡住提醒、完成记录和每日复盘串成闭环的个人执行驾驶舱。

## 功能列表

- 首页驾驶舱：今日统计、完成率、Top 3、本周重点、卡住提醒、17:00 复盘提醒、高价值输出与 SOP 候选统计。
- 任务收集箱：快速捕获碎片事项，按来源筛选，标记已整理，一键转为正式任务。
- 今日任务：新增、编辑、删除、完成、顺延到明天，按状态、优先级、四象限、类别筛选。
- 四象限：Q1/Q2/Q3/Q4 分区展示任务，预留拖拽扩展结构。
- 周任务：本周目标、进度条、状态管理，预留 related_task_ids。
- 重复任务：每天/每周/每月模板，一键生成今日任务，并避免同日同名重复生成。
- 完成记录：任务完成后自动写入，也支持手动新增，筛选高价值输出和 SOP 候选。
- 每日复盘：自动汇总今日完成、未完成、顺延和输出，支持评分与明日 Top 3。
- 设置：邮箱登录、退出、JSON 导入导出、手动同步、清空本地数据、通知权限、PWA 安装说明。
- AI 预留：`src/services/aiService.ts` 已预留分类、优先级、四象限、复盘和明日 Top 3 的规则模拟函数。

## 本地运行

```bash
npm install
npm run dev
```

未配置 Supabase 也可以直接使用，数据会保存在浏览器本地存储中，刷新不丢失。

## 打包

```bash
npm run build
```

打包产物在 `dist/`。

## Supabase 配置

1. 在 Supabase 创建新项目。
2. 打开 SQL Editor，执行 `supabase/schema.sql`。
3. 在 Authentication 中开启 Email 登录。Magic Link 登录可直接使用。
4. 复制 `.env.example` 为 `.env.local`，填入：

```env
VITE_SUPABASE_URL=你的 Supabase Project URL
VITE_SUPABASE_ANON_KEY=你的 Supabase anon public key
```

5. 重启开发服务。
6. 在应用设置页输入邮箱登录。登录后应用会先执行一次“云端 + 本地合并同步”，之后本地新增、编辑、完成、顺延、删除会自动推送到 Supabase。
7. 电脑端、手机端、网页端使用同一个邮箱登录后，即可共享同一份任务数据。另一台设备打开应用并登录同账号时，会自动合并云端数据。

所有云端表都包含 `user_id`，并通过 Row Level Security 限制用户只能访问自己的数据。

## 多端联动说明

- 未登录：纯本地模式，数据只在当前浏览器里。
- 已登录且在线：自动同步模式，增删改会同步到 Supabase。
- 已登录但离线：先写入本地，恢复网络后可在设置页点“立即合并同步”。
- 手机访问电脑本地开发服务时，电脑和手机需要在同一个 Wi-Fi 下，并用电脑局域网 IP 访问，例如 `http://192.168.1.23:5173`。

## 环境变量说明

- `VITE_SUPABASE_URL`：Supabase 项目 URL。
- `VITE_SUPABASE_ANON_KEY`：Supabase anon public key。

如果这两个变量为空，应用会自动进入本地离线模式。

## PWA 安装

- 电脑端：用 Chrome/Edge 打开应用，点击地址栏安装图标，添加到桌面。
- 手机端：用浏览器打开应用，选择“添加到主屏幕”。
- 应用包含 `manifest.json` 与 `service worker`，可离线打开最近缓存的应用外壳，本地数据保存在浏览器中。

## 数据导入导出

在设置页点击“导出 JSON”可保存当前本地数据。把 JSON 内容粘贴到导入框后点击“导入 JSON”，即可恢复到当前浏览器。
